# Interview Copilot — 文档生成提示词集

这个文件夹里放的不是文档本身，而是**用来生成三份面试复习文档的提示词**。把每个 `.md` 里的内容**整段复制**贴给 Codex / Claude / ChatGPT 等大模型，它们就会按要求把对应文档写出来。

|---|---|---|---|---|
| ① | **项目技术深度详解** | 通读后能完整理解整个项目的所有模块、链路、亮点 | [`01-project-technical-doc.md`](./01-project-technical-doc.md) | `docs/interview-prep/01-project-deep-dive.md` |
| ② | **面试问答库** | 应对面试时各种细节问题与追问，技术对比、参数取舍全覆盖 | [`02-interview-qa-bank.md`](./02-interview-qa-bank.md) | `docs/interview-prep/02-interview-qa-bank.md` |
| ③ | **简历追问问答** | 面试官针对简历每一句话可能挖的具体问题 | [`03-resume-defense.md`](./03-resume-defense.md) | `docs/interview-prep/03-resume-defense.md` |

> 三份都是面向"基础较薄弱、需要从零讲清概念"的读者，全中文、几乎不出现代码、用一步一步的流程描述 + 中文架构图。

## 怎么用

### 在 Claude Code / Codex CLI 里

1. `cd` 到本仓库根目录
2. 打开新会话
3. 把对应 `.md` 文件**全部内容**贴进去当输入
4. 让模型按提示词工作；它会读源码、写文档、保存到指定路径

### 在网页版 Claude / ChatGPT 里

1. 把对应 `.md` 文件**全部内容**贴进去
2. 模型没办法直接读你本地源码——所以请额外把提示词里 **"必读文件清单"** 列出来的源码内容也一起贴进去，或者：
3. 提前把这些文件打个 zip 一起上传（如果模型支持文件附件）

### 关键注意

- 提示词内部已经**预先梳理了大量项目事实**（4 大管道、记忆子系统、RAG 链路、评测指标等），即使模型读不到源码也能写出大致准确的文档；但**最高质量的输出仍来自能直接读源码的模型**（Codex CLI / Claude Code 这类有文件工具的）。
- 三份文档**互相独立**，可以按任意顺序生成，也可以并行。
- 文档②有"两阶段协作"流程（先出题、再答题），提示词里已经说明执行方式。

## 提示词的写作哲学

这三份提示词都遵循同一组原则，目的是让生成的文档对**基础薄弱的读者**真正可用：

1. **强制一步一步描述流程**：每个链路都要"当用户做了 X，系统首先……然后……接着……最终……"，每步说清"输入、做了什么、输出、为什么"。
2. **概念第一次出现必须从零讲解**：不假设读者懂 JWT、向量数据库、BM25、RAG、Reranker、SSE、Celery 等等。每一个出现的术语在第一次提到时必须用"白话"解释一遍。
3. **几乎不出现代码**：除非是 1-3 行点睛伪代码，全部用自然语言。
4. **每张架构图都要有中文注释**：不能只画英文盒子和箭头，要写"用户在前端点击发送 → 这一步会……"。
5. **亮点设计强制三段式**：① 这是干嘛的；② 怎么做到的（流程级，不写代码）；③ 为什么这样做（动机/权衡/否决的备选方案）。
6. **完整覆盖优先于篇幅**：可以不限字数，但绝不能漏任何模块或流程环节。

如果生成的文档没满足上面任何一条，可以让模型再迭代一轮，告诉它哪个章节不够细。

## 我们项目的事实速查（提示词里也都写了，这里给你自己对照）

**4 大业务管道**：
- **L1 QA 对话**：`/chat/sse/{session_id}` → SSE 流式 → 规划 → 并发召回（记忆+知识库）→ 生成
- **L2 ReAct Agent**：`/agent/react/stream` → QueryEngine → 工具循环 → trace 持久化
- **L3 模拟面试**：`/chat/mock-interview/start|answer|finish` → Runtime Director 三段式 LLM
- **L4 语音分析**：上传音频 → Celery 任务 → WhisperX 转写 → 三阶段 Map-Reduce 评分

**核心子系统**：
- **记忆子系统**（`services/memory/`）：`user_profile`（单文档补丁）+ `interview_fact`（多行表）；双阈值压缩；后台异步维护
- **RAG 子系统**（`rag/`）：向量 + BM25 混合 + Reranker + 分数阈值 + 词法兜底，三层 user_id 隔离
- **多模型抽象**（`core/model_registry.py`）：9 个 provider × `chat/agent/mock` 三个 role，per-user API key 加密存 DB
- **可观测**：LangSmith（全 LLM 调用）+ Sentry（错误）+ 自家 JSONL 指标

**指标基线**（简历里宣称、文档里需要验证一致）：
- 检索：Hit Rate@3 = 96.9%，MRR@5 = 0.950，P95 = 98ms（835 条样本）
- 生成（RAGAS 200 条）：忠实度 92.5%，上下文精确度 93.2%，上下文召回率 98%，E2E P95 ≈ 8s
